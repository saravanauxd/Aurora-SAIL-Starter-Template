# Contributing to Aurora SAIL Starter

Thank you for your interest in improving this template!

## How to Contribute

### Reporting Issues
- Use GitHub Issues for bugs or suggestions
- Include specific examples and error messages
- Describe your environment (Kiro version, OS, Node.js version)

### Suggesting Enhancements
- Open an issue describing the enhancement
- Explain the use case and benefits
- Provide examples if applicable

### Submitting Changes
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Test thoroughly in Kiro IDE
5. Submit a pull request with clear description

## What to Contribute

### Examples
Add SAIL interface examples to `examples/` directory:
- Dashboards
- Forms
- Work queues
- Reports
- Custom patterns

Include a brief description of what the example demonstrates.

### Documentation
Improve or expand:
- Setup instructions
- Troubleshooting guides
- Usage tips
- Best practices

### Patterns
Share reusable SAIL patterns by:
- Adding to `.kiro/steering/SAIL_SYNTAX_REFERENCE.md`
- Creating example files
- Documenting use cases

### Power Improvements
Enhance the Aurora Setup Power:
- Better error handling
- Additional validation
- Platform-specific improvements
- User experience enhancements

### Steering Files
Improve generation guidance:
- Common pitfalls to avoid
- Best practices
- Team-specific patterns
- Accessibility guidelines

## Guidelines

### Code Quality
- Test generated SAIL in Appian Designer
- Follow Aurora Design System conventions
- Keep examples minimal and focused
- Use clear, descriptive names

### Documentation
- Write for less technical users
- Include examples where helpful
- Keep instructions concise
- Update README if adding features

### Compatibility
- Maintain compatibility with Kiro IDE
- Test with current Aurora MCP version
- Don't break existing functionality
- Document any new requirements

## Testing Your Changes

Before submitting:

1. **Test setup process** - Run the Aurora Setup Power
2. **Generate SAIL** - Create sample interfaces
3. **Validate syntax** - Use the validation hook
4. **Test in Appian** - Verify code works in Designer
5. **Check documentation** - Ensure README is accurate

## Code of Conduct

- Be respectful and constructive
- Focus on improving the template for everyone
- Help others learn and succeed
- Welcome newcomers and answer questions

## Questions?

- Open a GitHub Issue for bugs or features
- Start a Discussion for questions or ideas
- Check existing issues before creating new ones

---

Thank you for helping make this template better for the Appian community!
